Zero-shot Chain-of-Thought (CoT)We can get a similar (or better) performance in a zero-shot setting•where we don’t provide any example questions •but simply pre-appending phrase:"Let’s think step by step.” •to the bot’s response:
see Kojima et al. “Large Language Models are Zero-Shot Reasoners”, 2022 https://arxiv.org/abs/2205.11916Moreover, if we take the analogy a little further and append phrase: “T ake a deep breath and work on this problem step-by-step.”•we see even better performance, despite fact that model is most certainly not breathing!see Yang et al., “Large Language Models as Optimizers”, 2023https://arxiv.org/pdf/2309.03409robot meditatinghttps://www.bing.com/images/create/robot-meditating/6512aa1e616a4acca642fdc7390b8a58

Self-consistency and Self-appraisalFor question answering, find answer that LLM is most confident about by:•sampling multiple outputs from LLM •thereby generating multiple responses to the question•count frequency of each distinct response generated•choose response with the highest frequency •since most common response is most likely to be rightOr simpler, get model to critique its own answer•provide information to user only when model agrees response was correct•otherwise try to regenerate it

Test-time Compute ScalingRecent models like Deepseek-R1 [Jan 2025]•trained to improve reasoning ability•by structuring output into two fields: <think> … </think> and <answer> … </answer>•model learns how much time to invest in “reasoning” about problem before committing to answerProcess is called test-time compute scaling•fine-tune on difficult questions requiring reasoning (e.g. maths)•dataset labelled with correct answerbut not reasoning required to get there •over training iterations, modellearns to produce longer responses,i.e. to spend more time ‘thinking’Can employ second (possibly bigger) model:•to check answer is correct, if ground truth not known •or to critique reasoning, e.g. force reasoning to be all in the same language
Source:   DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning, (Jan 2025)https://arxiv.org/pdf/2501.12948

Lack of RobustnessOngoing research question how to prompt LLMs to get reliable results•small changes in prompt can cause big changes to performance on some tasks•can be super frustrating!In general:•thebetter writtenthe prompt (i.e. the clearer and less ambiguous it is)  •the betterthe performanceTechniques for improving the prompt:•human: show prompt to person to see if they understand it! If not, improve it.•cross-validation: test the performance of different prompts on ground truth data•zero-shot: give prompt to a chatbot and ask it to improve the prompt
